## <span class="ez-toc-section" id="%E4%BB%80%E9%BA%BC%E6%98%AF_WSGI"></span>什麼是 WSGI?<span class="ez-toc-section-end"></span> 

Web Server Gateway Interface，的縮寫(Web伺服器閘道器介面)，是一種協議，這個協議制定了一套規則，規定HTTP Request要如何與Application Server溝通。

<ul class="wp-block-list">
  <li>
    WSGI應用是一個單調用、同步接口，即輸入一個請求，返回一個響應，無法支持長連接或者WebSocket
  </li>
</ul>

### <span class="ez-toc-section" id="WSGI_%E8%88%87_WSGI_Server"></span>WSGI 與 WSGI Server<span class="ez-toc-section-end"></span> 

可以將WSGI Server理解成處理 HTTP Request 與 Python 可理解的 Input/Output 的中繼站(Middleware)<figure class="wp-block-image">

<img decoding="async" src="https://i.imgur.com/JFpxNWj.png" alt="" /> </figure> 

所有支援 WSGI 協定的 Server 都可稱為 WSGI Server，現在比較常見的WSGI Server是gunicorn及 uwsgi。

## <span class="ez-toc-section" id="%E4%BB%80%E9%BA%BC%E6%98%AF_ASGI"></span>什麼是 ASGI?<span class="ez-toc-section-end"></span> 

<ul class="wp-block-list">
  <li>
    Asynchronous Server Gateway Interface(異步服務器網關接口)
  </li>
  <li>
    ASGI 是 WSGI 的繼承者，已經存在的WSGI應用可以直接在ASGI服務器中運行
  </li>
  <li>
    ASGI 代表異步服務器網關接口，是爲異步、同步應用程序提供標準，支持 WSGI 不支持當前 web 開發中的一些新的協議標準
  </li>
  <li>
    介於網絡協議服務和Python應用之間的標準接口，能夠處理多種通用的協議類型，包括HTTP，HTTP2和WebSocket。
  </li>
</ul>

## <span class="ez-toc-section" id="%E7%B8%BD%E7%B5%90_WSGI_%E8%88%87_ASGI"></span>總結: WSGI 與 ASGI<span class="ez-toc-section-end"></span> <figure class="wp-block-image">

<img decoding="async" src="https://i.imgur.com/0b6uf83.png" alt="" /> </figure> 

<ul class="wp-block-list">
  <li>
    相同之處:<ul>
      <li>
        WSGI 與 ASGI都指定接口並位於 Web 服務器和 Python Web 應用程序或框架之間。
      </li>
    </ul>
  </li>
  
  <li>
    兩者的區別:<ul>
      <li>
        WSGI是基於HTTP協議模式的，不支持WebSocket
      </li>
      <li>
        ASGI的誕生則是為了解決Python常用的WSGI不支持當前Web開發中的一些新的協議標準。
      </li>
      <li>
        ASGI對於WSGI原有的模式的支持和WebSocket的擴展，即ASGI是WSGI的擴展。
      </li>
    </ul>
  </li>
</ul>
