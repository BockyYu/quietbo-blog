Linux 安裝git

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ sudo apt-get install git</code></pre>

驗證git是否已安裝成功

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ which git</code></pre>

查詢git版本

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ git --version</code></pre>

git 產生SSH Key

<pre class="wp-block-code"><code lang="bash" class="language-bash">ssh-keygen</code></pre>

查看SSH Key

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ cat ~/.ssh/id_rsa.pub</code></pre>

查詢目錄的狀態

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ git status</code></pre>

查詢目前的設定

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ git config --list</code></pre>

查詢特定檔案的commit

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ git log XXXXX.py</code></pre>

注意:需要有副檔名

查看特定檔案修改什麼

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ git log -p XXXXX.py</code></pre>

+代表新增

-代表刪除

查詢&#8221;單行&#8221;代碼作者及修改時間

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ git blame XXXX.py</code></pre>

有些檔案不想放在Git裡面

先建立檔名為:

<pre class="wp-block-code"><code lang="bash" class="language-bash">.gitignore</code></pre>

然後編輯這個檔案的內容:

<pre class="wp-block-code"><code lang="bash" class="language-bash line-numbers">#忽略 test.py 檔案
test.py

#忽略 log目錄下的所有檔案
log/*

#忽略所有附檔名是 .tmp的檔案
.tmp

#忽略所有檔名為settings的檔案
settings.*</code></pre>

若不知道使用的工具或程式語言通常會忽略哪些檔案，可以至此[連結][1]查看

清出忽略

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ git clean -fx</code></pre>

忽略部分補充

如果要忽略的檔案在編輯.gitignore時就已經存在，需要先移出Git控管，之後就會被忽略了。

舉例:在3月1號時已經將test.py加入Git控管中，在3月5號才將.gitignore內新增要將test.py忽略，但此時Git已經在控管了，所以3月5號加的忽略不會進行，必須先將目前控管中的test.py移除。

退版方式

先查看版本號

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ git log --oneline</code></pre><figure class="wp-block-image">

<img decoding="async" src="https://i.imgur.com/UNXTYK1.png" alt="" /> </figure> 

方法一:相對位置

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ git reset d7ea967^</code></pre>

最後加的^代表前一次的意思，所以是指da824b4的commit，如果是d7ea967^^，就是前兩個版本，如果超過5個，可以寫成d7ea967~5，畢竟太多^很容易看得眼花。

相對位置也可以用另一種來表示:

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ git reset master^</code></pre>

或

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ git reset HEAD^</code></pre>

因為HEAD和master目前都是指向d7ea967這個commit，而且d7ea967這數字很難記，所以通常都不會用commit的英數來做，都已以上兩個方式，且都會一樣的結果。

方法二:絕對位置

直接指明在git log的版本號

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ git reset d9d68f9(版本號)</code></pre>

Reset 的模式

Reset英文單字翻譯為&#8221;重新設定&#8221;，但事實上Git的Reset指令用中文來說比較像是&#8221;前往&#8221;或&#8221;變成&#8221;的概念，事實上並沒有真的把這個commit&#8221;拆掉&#8221;(放心，所有commit都在)

比較正確的解讀應該是&#8221;我要前往兩個commit之前的狀態&#8221;，因為實際上git reset指令也不是真的刪除或是重新設定，只是&#8221;前往&#8221;到指定的commit。

git reset 指令可以搭配參數使用，常見的三種參數，分別:

<ul class="wp-block-list">
  <li>
    mixed
  </li>
  <li>
    soft
  </li>
  <li>
    hard
  </li>
</ul>

HEAD是什麼?

HEAD 是一個指標，指向某一個分支，通常你可以把 HEAD 當做「目前所在分支」看待。

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ cat .git/HEAD</code></pre><figure class="wp-block-image">

<img decoding="async" src="https://i.imgur.com/KlJ9Qat.png" alt="" /> </figure> 

從這個檔案看起來，HEAD 目前正指向著 master 分支。

如果有興趣再深入看一下 refs/heads/master 的內容就會發現，其實所謂的 Master 分支也不過就是一個 40 個字元的檔案罷了：<figure class="wp-block-image">

<img decoding="async" src="https://i.imgur.com/ZDlPkRN.png" alt="" /> </figure> 

切換分支，在看看HEAD檔案內容

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ git checkout dog
$ git cat .git/HEAD</code></pre><figure class="wp-block-image">

<img decoding="async" src="https://i.imgur.com/7CMHriu.png" alt="" /> </figure> 

HEAD 通常會指向目前所在的分支。不過 HEAD 也不一定總是會指向某個分支，當 HEAD 沒有指向某個分支的時候便會造成「detached HEAD」的狀態，詳情請參閱，待補。

在 SourceTree 的介面裡，HEAD 是以一個「空心的小圈圈」圖示呈現

在切換分支的同時，除了 HEAD 的內容會改變之外，在Reflog，當 HEAD 的內容改變的時候也會留下紀錄。

分支(Branche)

什麼是分支？

在開發軟體時，可能同時會有多人在開發同一功能或修復錯誤，也可能會有多個發佈版本的存在，並且需要針對每個版本進行維護。

為了能支援同時進行數個功能的增加或版本控制，Git具備了分支的功能。

建立分支

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ git branch 新的分支名稱</code></pre>

查詢現在的分支

<pre class="wp-block-code"><code lang="bash" class="language-bash">$ git branch</code></pre>

前方有*代表目前所在分支<figure class="wp-block-image">
