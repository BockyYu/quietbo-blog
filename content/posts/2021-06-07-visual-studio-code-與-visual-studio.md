我在開發web時使用VSCode，主要是寫前端、Python。(現在要多個C#了)  
Visual Studio是在大學、銀行專案和C++有使用過，這兩個我都有用過一段時間，  
最近發現，有些人把Visual Studio Code 和 Visual Studio誤會成同一個，可能是因為名稱和外型很像，所以會搞混，在這小說明這兩者之間的差異。

## Visual Studio Code {.wp-block-heading}<figure class="wp-block-image">

<img decoding="async" src="https://i.imgur.com/U9qsiXS.png" alt="" /> </figure> <figure class="wp-block-image"><img decoding="async" src="https://i.imgur.com/3QWvWax.png" alt="" /></figure> 

[Wiki:Visual Studio Code][1]  
下載點:[Visual Studio Code][2]  
Editor 編輯器  
輕量化、快速、著重在寫程式碼編輯，  
適用於Windows、MacOS及Linux，性能速度較快、可以各別擴充包且免費。  
<img decoding="async" src="https://i.imgur.com/QoMqC9k.png" alt="" /> 

事實上，我覺得VSCode不能與Visual Studio比較，能夠比較的對象應該是Sublime、Atom，因為這兩個都是Editor。

## Visual Studio {.wp-block-heading}<figure class="wp-block-image">

<img decoding="async" src="https://i.imgur.com/Jqy4iSQ.png" alt="" /> </figure> 

[Wiki:IDE 整合開發環境][3]  
[Wiki:Visual Studio][4]  
下載點:[Visual Studio][5]

IDE 可以說是編輯器、編譯、介面設計、測試、佈署的工具…等等，很多在開發時會用到的工具都放在裡面。  
屬於重量級的軟體。

下載的話是分Community、Professional及Enterprise。除了社區版以外都要付費，在安裝的過程中，套件或工具這些選擇越多，明顯感受到安裝越多時間越久，光是安裝就跟VSCode有明顯的落差。  
<img decoding="async" src="https://i.imgur.com/quwhDvG.png" alt="" /> 



小結:

對我來說Visual Studio Code 與 Visual Studio是不一樣的東西，  
所以並沒有特別覺得哪個較差，只是基於自己習慣和大多數人使用，還是VSCode較多。
