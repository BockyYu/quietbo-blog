下方有兩個進入shell的範例說明，  
方法一是建立container後直接進入shell，  
方法二是建立container後，再下指令進入運行中的container，我個人是很常用方法二。

docker Version: 20.10.7

## 方法一 

創建ubuntu(或busybox)的container並進入shell

<pre class="wp-block-code"><code lang="docker" class="language-docker">docker container run -it ubuntu sh  
或
docker container run -it busybox sh</code></pre>

用法：

<blockquote class="wp-block-quote is-layout-flow wp-block-quote-is-layout-flow">
  <p>
    docker container run [OPTIONS] IMAGE [COMMAND] [ARG…]
  </p>
</blockquote>

<ul class="wp-block-list">
  <li>
    -it ：啟動容器並進入終端交互模式(it通常是一起使用的)
  </li>
</ul>

想查看更多_OPTIONS_可輸入

<pre class="wp-block-code"><code lang="docker" class="language-docker">docker container run --help</code></pre>

進入某的運行中的容器裡面

<pre class="wp-block-code"><code lang="docker" class="language-docker">docker exec -it CONTAINER_ID sh</code></pre>

退出方式有兩種：

<ol class="wp-block-list">
  <li>
    exit 退出容器後，容器會停止
  </li>
  <li>
    ctrl+P+Q 退出容器但容器不停止
  </li>
</ol>

## 方法二 

建立nginx的container

<pre class="wp-block-code"><code lang="docker" class="language-docker">docker container run -d  -p 80:80 nginx</code></pre>

查詢CONTAINER ID，輸入指令進到container的內部。

<pre class="wp-block-code"><code lang="docker" class="language-docker line-numbers">docker ps
docker exec -it CONTAINER_ID sh</code></pre>

用法：

<blockquote class="wp-block-quote is-layout-flow wp-block-quote-is-layout-flow">
  <p>
    docker exec [OPTIONS] CONTAINER COMMAND [ARG…]
  </p>
